# คู่มือการตั้งค่า `rate_targets` (อัตราที่ติดตาม)

เอกสารนี้อธิบายวิธีกำหนดว่าจะให้ระบบอ่าน "อัตราตัวไหน" จาก PDF ประกาศของแต่ละธนาคาร
ตั้งค่าได้ 2 ทาง — ผลเหมือนกัน:

- **หน้าเว็บ `/config`** (แนะนำ; admin เท่านั้น) — แก้แล้วกด "บันทึกการตั้งค่า"
- **แก้ไฟล์ `data/banks_config.json`** ตรง ๆ

> เมื่อบันทึก config การ parse ที่ cache ไว้ของธนาคารนั้น **หมดอายุเอง** (ผ่าน `targets_signature`)
> รอบ `--backfill` ถัดไปจะอ่าน PDF ใหม่ให้ ค่าจึงจะปรับตาม config — ดูหัวข้อ "cache & การมีผล" ท้ายเอกสาร

---

## 1. แนวคิด: 1 target = 1 อัตราที่ติดตาม

`rate_targets` เป็น **รายการ (list)** ของ "เป้าหมายอัตรา" ของธนาคารหนึ่ง แต่ละ target คือค่าดอกเบี้ย
หนึ่งค่าที่เราสนใจ (เช่น "ประจำ 12 เดือน วงเงิน 1 ล้าน ของบุคคลธรรมดา")

- แต่ละ target สร้าง **2 คอลัมน์ใน CSV**: `<key>` (ค่าอัตรา) และ `change_<...>` (ส่วนต่างจากประกาศก่อน)
- แต่ละ target = **1 เส้นในกราฟ** และ **1 แถวในตาราง** บนหน้า `/bank/{code}`
- **ลำดับของ target = ลำดับคอลัมน์ใน CSV** (เพิ่ม target ใหม่ → คอลัมน์ต่อท้าย)
- เพิ่ม target ใหม่ **ไม่ต้องแก้โค้ดฝั่งเว็บเลย** — กราฟ/ตารางไล่อ่านจาก `rate_targets` อัตโนมัติ

---

## 2. ฟิลด์ของแต่ละ target

| ฟิลด์ | จำเป็น | ความหมาย |
|---|---|---|
| `key` | ✅ | ชื่อคอลัมน์/ตัวระบุ (ห้ามซ้ำในธนาคารเดียวกัน, ห้ามว่าง) เช่น `rate_12m_1m`, `saving` |
| `tenor_months` | ✳️ | ระยะฝาก (เดือน) เป็นจำนวนเต็ม — ใช้กับ **เงินฝากประจำ**; ผลิตภัณฑ์ที่ไม่มีระยะ(ออมทรัพย์) ใส่ `null` |
| `row_keyword` | ✳️ | คำ/วลีที่อยู่ใน "ชื่อแถวผลิตภัณฑ์" ของ PDF — ใช้กับผลิตภัณฑ์ **ที่ไม่มี tenor** หรือระบุแถวเอง |
| `section_keyword` | — | คำในหัวข้อ "หมวด" เพื่อจำกัดขอบเขตการค้นหาแถว (ดูตารางข้อ 3 ว่าแต่ละธนาคารใช้ไหม) |
| `amount_m` | — | วงเงินที่ใช้เลือก tier — **หน่วยล้านบาท** (ดูข้อ 4 เรื่องหน่วยแสน) เว้นว่าง = ใช้ tier แรกที่พบ |
| `depositor` | — | ประเภทผู้รับดอกเบี้ย (คีย์เวิร์ดหรือเลขคอลัมน์) — เว้นว่าง = บุคคลธรรมดา (ดูข้อ 3) |
| `label` / `alias` | — | ชื่อที่แสดงบนกราฟ/ตาราง (`alias` มาก่อน `label` มาก่อน `key`) |

✳️ **กติกาบังคับ (validation):** ทุก target ต้องมี **`tenor_months` หรือ `row_keyword` อย่างน้อยหนึ่งอย่าง**
ไม่งั้นระบบหาแถวใน PDF ไม่ได้ (หน้า `/config` จะไม่ยอมบันทึก)

## 3. ค่าเริ่มต้น/ข้อจำกัดต่อธนาคาร

วิธีอ่าน PDF ของแต่ละธนาคารต่างกัน จึงมีค่าเริ่มต้น/ข้อจำกัดต่างกัน:

| ธนาคาร | วิธีอ่าน | แถวประจำ (ค่าเริ่มต้นเมื่อใส่ `tenor_months`) | `section_keyword` | คอลัมน์ `depositor` ที่รองรับ | รูปแบบ tier วงเงิน |
|---|---|---|---|---|---|
| **SCB** | ข้อความ (13 คอลัมน์) | `{N} เดือน` | **บังคับ** (ค่าเริ่มต้น "เงินฝากประจำ แบบมีสมุด") | 1–13 (ดูข้อ 3.1) | น้อยกว่า / ตั้งแต่…ขึ้นไป |
| **KTB** | ข้อความ (10 คอลัมน์) | `ประจำ {N} เดือน` | ไม่บังคับ | 1–10 | ช่วง "ตั้งแต่ X ถึง Y" + น้อยกว่า/ขึ้นไป |
| **KBANK** | พิกัด x (extract_words) | `เงินฝากประจำ {N} เดือน` | **ไม่ใช้** (ใส่ไปก็ไม่มีผล) | บุคคลธรรมดา, ราชการ, สถาบันการเงิน, กองทุน เท่านั้น | X.0 ล้านบาท + **แสน (ไม่เกิน/ส่วนที่เกิน)** |
| **BAY** | ข้อความ (11 คอลัมน์, ตัด cid) | `ระยะเวลาการฝาก {N} เดือน` | **บังคับ** (ค่าเริ่มต้น "บัญชีเงินฝากประจำ") | 1–11 | น้อยกว่า / ช่วง / ขึ้นไป (แสน+ล้าน) |
| **BBL** | OCR (tesseract, 9 คอลัมน์) | แถว `ระยะเวลาการฝาก {N} เดือน` ในหมวด `ประจำ` | ไม่บังคับ (ค่าเริ่มต้น "ประจำ") | 1–9 | above/up_to/less_than/at_least |

### 3.1 เลขคอลัมน์ `depositor` ของ SCB
1=บุคคลธรรมดา 2=นิติบุคคลทั่วไป 3=ราชการ 4=นิติบุคคลไม่แสวงหากำไร 5=สถาบันการเงิน
6=สถานศึกษา 7=กองทุน 8=สหกรณ์ 9–11=ผู้มีถิ่นฐานนอกประเทศ 12=นิติบุคคลพิเศษ 13=บุคคลพิเศษ

> ระบุ `depositor` เป็นคีย์เวิร์ด (เช่น `"กองทุน"`, `"ราชการ"`) ได้ทุกธนาคาร ระบบจะแปลงเป็นคอลัมน์ให้เอง
> ถ้าคีย์เวิร์ดกำกวม/หาไม่เจอ ใช้ **เลขคอลัมน์** ตรง ๆ แทนได้ (สำหรับ SCB/KTB/BAY/BBL)

---

## 4. หน่วยของ `amount_m` — วงเงินหน่วยแสนใช้ทศนิยม

`amount_m` มีหน่วยเป็น **ล้านบาท** เสมอ วงเงินหน่วย "แสน" ให้ใส่เป็นทศนิยม:

| วงเงินจริง | `amount_m` |
|---|---|
| 5 แสนบาท | `0.5` |
| 3 แสนบาท | `0.3` |
| 1 ล้านบาท | `1` |
| 10 ล้านบาท | `10` |

ระบบจะเลือก tier ที่วงเงินนี้ตกอยู่ (เช่น `0.3` เข้ากับ tier "ไม่เกิน 5 แสนบาท"; `0.7` เข้ากับ "ส่วนที่เกิน 5 แสนบาท ขึ้นไป")

## 5. ตัวอย่างครบทุกสถานการณ์

### 5.1 เงินฝากประจำมาตรฐาน (มี tenor)
```json
{ "key": "rate_12m_1m", "tenor_months": 12, "amount_m": 1,
  "label": "12 เดือน / วงเงิน 1 ล้านบาท", "alias": "12 เดือน / วงเงิน 1 ล้านบาท" }
```

### 5.2 ออมทรัพย์ (ไม่มี tenor — ใช้ `row_keyword`)
`tenor_months: null` + `row_keyword` ชี้ชื่อแถวผลิตภัณฑ์ (คำที่แต่ละธนาคารใช้ต่างกัน)
```json
{ "key": "saving", "tenor_months": null, "amount_m": 1, "row_keyword": "ออมทรัพย์",
  "label": "ออมทรัพย์ / 1 ล้านบาท", "alias": "ออมทรัพย์ / 1 ล้านบาท" }
```
> ตัวอย่างคำที่ใช้จริง: KBANK/KTB/BAY = `"ออมทรัพย์"`, BBL = `"สะสมทรัพย์"`,
> SCB = `section_keyword: "ออมทรัพย์แบบมีสมุดคู่ฝาก"` + `row_keyword: "ออมทรัพย์"`

### 5.3 หัวข้อผลิตภัณฑ์ยาว 2 บรรทัด และไม่มีระยะฝาก — **เคสจริง MAKE by KBank**
บางผลิตภัณฑ์มีชื่อยาวจน PDF ตัดขึ้น 2 บรรทัด และบรรทัดแรกซ้ำกับผลิตภัณฑ์อื่น
(เช่น "เงินฝากออมทรัพย์อิเล็กทรอนิกส์ (K-eSavings)" ที่ทั้ง K-eSavings ธรรมดาและ MAKE ใช้เหมือนกัน)
ต้องแยกด้วย **บรรทัดที่สอง** "ผ่านบริการ MAKE by KBank"

**วิธี A — ใส่ชื่อเต็มทั้ง 2 บรรทัด** (ระบบจับแบบต่อบรรทัดให้เอง):
```json
{ "key": "make_saving", "tenor_months": null, "amount_m": 0.3,
  "row_keyword": "เงินฝากออมทรัพย์อิเล็กทรอนิกส์ (K-eSavings) ผ่านบริการ MAKE by KBank",
  "label": "MAKE by KBank / ≤5 แสนบาท", "alias": "MAKE by KBank / ≤5 แสน" }
```

**วิธี B — ใช้เฉพาะคีย์เวิร์ดที่ "ไม่ซ้ำใคร" ในบรรทัดที่สอง** (สั้นและชัวร์กว่า):
```json
{ "key": "make_saving", "tenor_months": null, "amount_m": 0.3,
  "row_keyword": "ผ่านบริการ MAKE by KBank",
  "label": "MAKE by KBank / ≤5 แสนบาท", "alias": "MAKE by KBank / ≤5 แสน" }
```
> ทั้งสองวิธีอ่านได้ `1.25%` จากประกาศ 2026-05-09 (tier "ไม่เกิน 5 แสนบาท" เพราะ `amount_m: 0.3`)

**กับดักที่ควรรู้:** ระบบพยายามจับ **ทีละบรรทัดก่อน** ถ้าไม่สำเร็จค่อยลองต่อ 2 บรรทัด
ถ้าตั้ง `row_keyword` เป็นวลีที่ **บังเอิญไปตรงกับบรรทัดในส่วน "เงื่อนไข" ท้ายเอกสาร**
(เช่น "…MAKE by KBank มีเงื่อนไขดังนี้") ระบบอาจไปเจอบรรทัดนั้นก่อน — จุดนั้นไม่มีตารางอัตรา ระบบจะข้ามแล้วลองต่อ
2 บรรทัดให้เอง แต่ถ้าอยากชัวร์ ให้ใช้ **วิธี B** (คีย์เวิร์ดเฉพาะบรรทัดสอง)

### 5.4 ระบุประเภทผู้รับดอกเบี้ย (`depositor`)
```json
{ "key": "rate_12m_10m_gov", "tenor_months": 12, "amount_m": 10, "depositor": "ราชการ",
  "label": "12 เดือน / 10 ล้านบาท / ราชการ", "alias": "12 เดือน / 10 ล้านบาท / ราชการ" }
```
> KBANK ใช้ `"หน่วยงาน ราชการ"`; ธนาคารอื่นใช้ `"ราชการ"` — หรือใส่เลขคอลัมน์แทนได้ (SCB/KTB/BAY/BBL)

### 5.5 วงเงินหน่วยแสน
```json
{ "key": "make_saving", "tenor_months": null, "amount_m": 0.5,
  "row_keyword": "ผ่านบริการ MAKE by KBank", "label": "MAKE / 5 แสน", "alias": "MAKE / 5 แสน" }
```

### 5.6 ผลิตภัณฑ์บรรทัดเดียวที่มีค่าอยู่ในหัวข้อ (เช่น LINE BK)
บางผลิตภัณฑ์มีอัตราเดียวไม่มี tier วงเงิน — ใส่ `row_keyword` ชี้ชื่อแถว ระบบอ่านค่าจากแถวนั้นเลย
```json
{ "key": "linebk", "tenor_months": null, "row_keyword": "เงินฝาก LINE BK",
  "label": "LINE BK", "alias": "LINE BK" }
```

## 6. cache & การมีผล / ข้อผิดพลาดพบบ่อย

- **การมีผล:** บันทึก config → `targets_signature` เปลี่ยน → parse cache ของธนาคารนั้นหมดอายุ →
  รอบ `--backfill` ถัดไป (หรือกดปุ่ม "เติมข้อมูลย้อนหลัง" บนหน้า `/bank`) อ่าน PDF ใหม่ ค่าจึงปรับ
- **เพิ่ม `key` ใหม่:** คอลัมน์ CSV ถูกเพิ่มและ **เติมย้อนหลังจาก PDF ที่เก็บไว้** ให้อัตโนมัติ
  ไฟล์เก่าที่ยังไม่มีผลิตภัณฑ์นั้นจะเว้นว่าง + ขึ้น log "ข้าม target" — เป็นเรื่องปกติ ไม่ใช่บั๊ก
- **ข้อผิดพลาดพบบ่อย:**
  - `key` ซ้ำ/ว่าง → บันทึกไม่ได้
  - ไม่มีทั้ง `tenor_months` และ `row_keyword` → บันทึกไม่ได้
  - `depositor` ที่ KBANK ยังไม่รองรับ (เช่น นิติบุคคล(1)/(2)) → target นั้นถูกข้าม
  - `row_keyword` กว้างเกินจนไปตรงบรรทัดเงื่อนไขท้ายเอกสาร (ดูข้อ 5.3) → ใช้คีย์เวิร์ดเฉพาะบรรทัดสอง

## 7. อัปโหลดประกาศเอง (เมื่อดาวน์โหลดจากเว็บธนาคารไม่ได้)

ถ้าระบบดาวน์โหลด PDF ประกาศจากเว็บธนาคารอัตโนมัติไม่ได้ (เว็บบล็อกบอต/URL เปลี่ยน) หรือมีไฟล์
ประกาศย้อนหลังที่ระบบยังหาไม่เจอ — อัปโหลดไฟล์เองได้จากหน้า `/bank/{code}` (admin เท่านั้น):

1. กดปุ่ม **⬆ อัปโหลดประกาศ** ในทูลบาร์
2. เลือกไฟล์ PDF — **วันที่มีผล** เว้นว่างได้ ระบบจะอ่านจากเนื้อไฟล์ให้ (อ่านไม่ได้ค่อยกรอกเอง `YYYY-MM-DD`)
3. ระบบเซฟไฟล์ลง `pdfs/<CODE>/<code>_deposit_<วันที่>.pdf` แล้วสั่ง "เติมข้อมูลย้อนหลัง" ให้เอง

**ข้อควรรู้:**
- **ชื่อไฟล์ = วันที่ประกาศ** เป็นตัวอ้างอิงของประวัติ — ถ้าอัปโหลดไฟล์ซ้ำวันเดิม ระบบจะถามยืนยัน
  ก่อนเขียนทับ (ไม่ทับเงียบ ๆ)
- ถ้ามีงานอื่นค้างอยู่ตอนอัปโหลด ไฟล์จะถูกเซฟแล้วแต่ CSV ยังไม่อัปเดตจนกว่าจะกด "เติมข้อมูลย้อนหลัง" อีกครั้ง
- อัปโหลด **ผิดไฟล์/ผิดธนาคาร**: ไฟล์จะ parse ไม่ได้ (ช่องว่าง/แถบเตือน) — แก้โดย **ลบไฟล์นั้นออกจาก
  `pdfs/<CODE>/`** แล้วกด "เติมข้อมูลย้อนหลัง" ใหม่
- อัปโหลดได้เฉพาะธนาคารที่ `enabled: true`
